# 00 · 總覽（Overview）

## 這是什麼

一個 **TronClass 校園系統的跨平台原生客戶端**，自動化兩件事：

1. **點名簽到（rollcall）**——偵測課堂上老師發起的點名，替登入的使用者完成簽到。
2. **測驗答題（auto-answer）**——偵測進行中的測驗／作業，用 LLM 備妥答案，經使用者確認後送出。

單一 UI codebase，跑 **Windows / macOS / Linux / Android / iOS** 原生。桌面與 Android 為主場，
iOS 因平台限制為二等公民（見下）。

## TronClass 是什麼

TronClass 是一套校園 LMS（學習管理系統）。**各校自取品牌名**——iLearn、iClass、TronClass、
各種校名前綴都有——但**底層是同一套 HTTP API**。因此本客戶端只要知道一所學校的 `base_url`，
其餘端點都能推導（見 `40-providers.md`）。學生用它上課、看教材、考試、被點名；本專案就是這個
學生端經驗的自動化替身。

## 目標平台與 iOS 的誠實分級

同一份 MAUI UI code 編譯到五個平台（Linux 經 Avalonia MAUI backend 算繪，見 `10-architecture.md`）。
**按鈕與畫面在每個平台都一樣存在**，差別只在某些能力 iOS 給不了：

| 能力 | Windows / macOS / Linux / Android | iOS |
|---|---|---|
| 前景操作（開著 App 手動觸發） | ✅ | ✅ |
| 背景監控（螢幕關著自動簽到/答題） | ✅（Android 前景服務） | ❌ 只能前景 |
| 應用內自動更新 | ✅（桌面/APK） | ❌ sideload 無法自更新 |
| 易安裝 | ✅ | ⚠️ 需 sideload 工具、免費證 7 天過期 |

iOS 不砍功能、不砍畫面——它只是**在能力旗標上把做不到的項目回報為不可用**（見 `20-contract.md`
的 capability flags）。UI 依旗標 enable/disable，**永不寫 `if iOS`**。**絕不對外把各平台講成對等。**

## 北極星：精巧優雅

**Compact and elegant.** 沒有臃腫、沒有投機抽象、能不加的依賴就不加。一個功能、一層抽象、一個
依賴，若賺不到它的存在，它就不存在。這是硬要求，貫穿每一份規格與每一行實作。當「多做一點以防
萬一」與「精簡」衝突時——**精簡勝出**，除非那是輸入驗證、錯誤處理、資安邊界或硬體/協定校準
（這四類不在精簡之列）。

## Non-goals（明確不做）

- **不架伺服器。** 本客戶端在使用者自己的裝置上跑，直接對學校 API。不做「幫使用者輪詢的雲端後端」
  ——那會把帳密/session 挪到伺服器、破壞信任模型、且是臃腫。（若未來要「常駐 runner 遙控」也只能是
  使用者自架、opt-in 的進階選項，非預設。）
- **不上架 App Store / Play Store。** 只在 GitHub 發佈 apk / ipa。
- **WASM/瀏覽器目標延後。** Avalonia backend 順帶能出 WASM，但 v1 不做。
- **手動 QR 貼上/掃描不是「自動」路徑。** 見 `32-domain-qr.md`：唯一自動 QR 路徑是教師輔助；任何
  手動輔助工具都必須誠實標為「手動」，**永不包裝成自動**。
- **不做為了未來而先搭的鷹架。** 需要時再長（見 `50-build-order.md` 的增量分片）。

## 定位與授權情境

這是一個**個人向的自動化工具**：使用者對**自己的學校、自己的帳號**操作。開發與驗證一律在使用者
自有的測試帳號與測試課程上進行。授權採 **AGPL-3.0-or-later**（出於開源信念的主動選擇，見 `LICENSE`）。

本專案是一次**全新的 clean-room 實作**：`docs/` 是規格（要做到什麼／伺服器怎麼運作），不是任何既有
程式的描述。用你自己的方式，在 Rust + .NET MAUI 上寫出優雅的實作。
