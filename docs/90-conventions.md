# 90 · 硬規矩（Conventions）

違反這些會被打回。多數源自長期教訓。每條附上 why。

## 1. Clean-room：從規格實作，不繼承任何積垢

`docs/` 是規格（要做到什麼／伺服器怎麼運作），**不是**任何既有程式的描述。**不要去找、不要抄任何
先前實作**，也不要把任何舊模組名/函式名當作實作藍圖。
**Why**：本專案的優雅，正是靠不繼承別人邊做邊縫的結構。用你自己的方式在 Rust + MAUI 上重新設計。

## 2. 學校一律平等

code / README / UI / 設定 四處都「**全列或都不列**」；任何名單一律**從 registry 生成**（`40-providers.md`），
**禁止任何一處手打某校字面值**。能力用統一預設，不寫進 per-school 資料。登入分支**以協定/頁面特徵命名，
絕不以學校命名或分支**（`30`）。
**Why**：避免對某校特殊待遇、避免名單漂移；這是專案的長期立場。

## 3. QR 誠實（見 `32-domain-qr.md` 全文）

純學生端偽造 `data` **尚未發現（非不可能）**——永不寫「不可能／不可偽造」。聽聞的成功者一律稱
「傳聞／都市傳說」，**永不指名真人**。手動路徑誠實標「手動」，**永不包裝成自動**。QR 的負面結果地圖
公開保留、別刪。
**Why**：對外誠實是專案信譽；把「尚未發現」講成「不可能」既不誠實也會誤導後繼研究。

## 4. 祕密絕不落明碼 + 全面遮蔽

帳密、LLM 金鑰等祕密**只進加密保險庫**（`10-architecture.md`），**永不明碼落地**。所有 log / status /
匯出 / debug 輸出一律經**單一遮蔽走訪**把祕密遮成 `[redacted]`。
**Why**：「會不會漏祕密」要能只審一處；單一遮蔽點 + 加密庫讓這件事可稽核。
**別**在寫檔路徑遮蔽（那會把使用者真正的金鑰/密碼寫成 `[redacted]` 毀檔）——只在顯示/序列化層遮。

## 5. 錯誤永不靜默

登入失敗、LLM 連不上、送出被伺服器擋——一律成 `Error` 事件（帶可讀原因）上到 UI。
**LLM 失敗時寧可不送，也不送空白答案。**
**Why**：曾有「準備答案中…」卡住卻無提示的教訓；靜默吞錯讓使用者無從得知。

## 6. iOS 二等公民，誠實交代

同一份 UI code，但背景監控、自動更新、易安裝在 iOS 上做不到。用**能力旗標**暴露可用性，UI 依旗標
enable/disable，**永不寫 `if iOS`**，也**永不對外把各平台講成對等**。
**Why**：平台限制是事實；假裝對等會讓使用者誤判。

## 7. 精簡優先（YAGNI），但四類不簡化

能不加的依賴/抽象/檔案就不加；投機需求先跳過。**但輸入驗證、錯誤處理、資安邊界、硬體/協定校準
（雷達座標、時間戳、伺服器逐字比對之類）不在簡化之列。** 刻意留的捷徑要在碼裡標明上限與升級路徑。
**Why**：臃腫是本專案要極力避免的；但把安全/正確性簡化掉是另一種債。

## 8. 雷達求解器零外部數學套件

WGS84 多點定位用純運算實作，**不引入 numpy/scipy 等數學套件**。
**Why**：可打包、精簡；一個定位功能不值得拖一整包數學依賴。

## 9. 送出值逐字正確（見 `31` 計分 gotcha）

填空/簡答連 HTML 標記一起送（送純文字＝0 分）；matching 用成員驗證避免誤配。顯示層的清理**絕不**污染
送出值。
**Why**：伺服器逐字比對；顯示好看與送出正確是兩件事，混淆會靜默丟分。

## 10. Core 離線可測

core 完全 headless，對一個**假 TronClass 伺服器**用命令驅動、斷言事件，覆蓋登入/四型點名/答題全流程；
純決策邏輯（答題決策、雷達解算）直接單測。非平凡邏輯至少留一個可跑的檢查。
**Why**：不碰真實學校即可回歸；把 UI 動盪與領域正確性隔開。

## 11. 工具鏈與風格

- **Rust core**：用 `rustfmt` + `clippy`（warnings 視為錯誤）——這是 Rust 生態的原生紀律，採用它。
- **.NET UI**：用標準 analyzers；為 **Native AOT** 而寫（避免反射式模式）。
- 新碼比照周邊風格。
**Why**：clean-room 專案該用生態的標準工具，不重蹈「無 formatter、全手動比照」的舊苦。

## 12. 給足必要、withhold 機密

規格要**給足**冷啟動實作所需的一切技術資訊；但**真實帳密/金鑰/擷取到的 token/真實 payload/任何祕密**
一個字都不放進 repo。
**Why**：交接要完整，但機密外洩不可逆。

## 13. 授權情境

這是使用者對**自己的學校、自己的帳號**操作的個人工具。開發與驗證一律在使用者自有的測試帳號與測試
課程上進行。授權 **AGPL-3.0-or-later**（主動選擇，見 `LICENSE`）。
