# 60 · UI 資訊架構（Information Architecture）

> **本檔只定「畫面結構／導覽」，不定視覺。** 版面、字體、動效、配色留待之後一個**專門的 UI 設計
> 階段**；現在的 UI 是丟棄式最小實作，只為驅動 core（見 `50-build-order.md`）。這份是那個階段的
> 結構規格：畫面有哪些、彼此怎麼走、每個畫面的職責。長相不在此規範內（刻意）。

## 底部分頁（四個）

**首頁 / 點名 / 答題 / 帳號**。設定**不是**第五個分頁——它是「帳號」分頁裡的一個入口點開的設定頁。

### 首頁（Home）
- **監控總開關**（開始／停止監控，對應 `StartMonitoring`/`StopMonitoring`）。
- **被監控帳號摘要**：目前納入監控的帳號、各自線上／登入失敗狀態（`AccountStatus` 事件）。
- **目前狀態**：idle／monitoring／offline（`StateChanged`）。
- **下一堂課**：來自 courses 端點（`/api/my-courses`）。
- **近期活動**：最近偵測到的點名／答題摘要。
- **背景可用性**：依 capability flag（`background_monitoring`，見 `20-contract.md`）感知——iOS 顯示「僅前景」。

### 點名（Rollcall）
- **近期點名紀錄** + **目前開放中的點名清單**（合併後的活動，一活動一列，`RollcallDetected`）。
- 點一項 → **該點名詳細頁**（型別、課程、全班簽到率、參與帳號、倒數、各帳號簽到狀態）。

### 答題（Answer）
- 同「點名」的形狀：近期紀錄 + 進行中清單 → 點進去是**答題專屬頁**。（答題領域見 `31`，屬後續 slice。）

### 帳號（Accounts）
- 同時**監控多個帳號**（可跨班級、跨機構），各自新增／切換／刪除。
- 登入失敗時可改走「**瀏覽器 cookie 登入**」後備（core 端 `ImportCookies` 已支援；擷取畫面屬本階段之後）。
- **設定入口**在此（點開設定頁：倒數秒數、15% 門檻、LLM/供應端等）。

## 快速彈窗（Quick popup）

**定義：一個置中的小 modal，供即時操作。** 在有時限流程觸發時彈出：
- **點名**：立即簽（`SignNow`）／暫緩（`DeferSignIn`）。
- **答題**：送出（`SubmitNow`）／暫緩（`HoldAnswer`）／捨棄（`DiscardAnswer`）。
- 彈窗上的「**詳細**」→ 導向「點名」或「答題」分頁中對應的那個活動。

## 導覽關係（一句話）

四分頁平行；「首頁」是總覽與總開關，「點名／答題」是各自領域的清單→詳細，「帳號」管帳號與設定；
有時限操作一律先走快速彈窗，要看全貌再進對應分頁。所有資料來自 core 事件、所有動作送 core 命令
（見 `20-contract.md`）——UI 零領域邏輯。
